## 跑你的第一份体检

最常见:你拿到一个潜在客户的网址 `https://example.com`:

```bash
node bin/cli.js https://example.com
```

如果对方的站是 React/Vue 那种打开慢、刷一下才出内容的(单页应用 SPA),加 `--render`:

```bash
node bin/cli.js https://example.com --render
```

跑完会看到:

```
✓ 体检完成，用时 25 秒
  页面：47   问题：86 处（23 类）· P0=3  P1=22  P2=55  P3=3
  📄 HTML 报告：report/体检报告.html
```

双击打开 `report/体检报告.html` 看效果。把 `report/` 整个文件夹发给客户就行。

### 几种常见情况

| 客户的站 | 命令 |
|---|---|
| 普通 Shopify / WordPress | `node bin/cli.js https://xxx.com` |
| 打开要刷一下才出内容(SPA) | `node bin/cli.js https://xxx.com --render` |
| 想抓深一点 | `node bin/cli.js https://xxx.com --render --max-pages 100` |
| 报告单独放一个客户文件夹 | `node bin/cli.js https://xxx.com --out ./客户A` |

> 判断要不要加 `--render`:先不加跑一次,如果很多页「字数很少、缺 H1」,或只抓到 1 页,就加 `--render` 重跑。

## 把报告发给客户(你的成交动作)

1. 打开 `report/体检报告.html`,先自己扫一眼 **P0 那几条**(致命问题)—— 这是你跟客户开口的由头。
2. 把 `report/` 文件夹发给客户;或在浏览器里「打印 → 存为 PDF」做成更正式的版本。
3. 客户读报告 → 看到一堆红色 P0/P1 → 翻到底部 → 扫你的二维码 → 你接住。

报告里**最有杀伤力的两块**:
- **问题清单**:P0「没有 sitemap / 主域打不开 / SPA 空壳」这种,客户一看就慌。
- **Fan-Out 候选 query** 里大量 ✗(未覆盖)—— 直接告诉客户「这些该搜到你的词,你一个都没占」。

## 常见问题

**Q:跑 `--render` 报错 "No Chrome found"?**
装一下 Google Chrome,或在命令后加 `--chrome "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"`。

**Q:只抓到 1 个页就停了?**
多半是 SPA,加 `--render` 重跑。

**Q:报告底部二维码是个空框?**
你还没放 `qr-wechat.png`,回到第 3 步。

**Q:客户问「为什么 Lighthouse 给我 100 分,你却说我有问题」?**
Lighthouse 只看单页「标签齐不齐」,看不到跨页问题(title 全站重复、没 sitemap、关键词内耗)。给客户看报告 P0 那一栏的具体问题就行。

**Q:客户的数据会上传到哪吗?**
不会。全程在你电脑本地跑,只对目标站发请求,数据不出本机。
