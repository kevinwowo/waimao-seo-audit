# 外贸独立站 SEO 体检工具 · 使用指南

本指南面向**不会写代码也能跑**的外贸 SEO 运营。如果你能装个 App、能复制粘贴命令,就能用它。

全程 5–10 分钟,免费,不上传任何数据。

---

## 一、第一次准备(只需做一次,5 分钟)

### 1. 装 Node.js

去 https://nodejs.org/ 下载 **LTS 版本(20 或更高)**,一路下一步装好。

装好后打开**终端**(Mac:聚焦搜索 → "终端";Windows:开始 → cmd 或 PowerShell),输入:

```bash
node -v
```

看到类似 `v20.x.x` 就说明装好了。

### 2. 下载本工具

```bash
git clone https://github.com/kevinwowo/waimao-seo-audit.git
cd waimao-seo-audit
npm install
```

(没装 git 也可以直接下载 ZIP,解压后在文件夹里打开终端。)

`npm install` 跑完,工具就装好了。**这一步只做一次。**

### 3. 填上你的联系方式

用任何文本编辑器打开 `contact.json`,把方括号里的占位文字换成你自己的:

```json
{
  "brand": "张三 / 跨境 SEO 工作室",
  "tagline": "外贸独立站 SEO 顾问 · 跨境电商增长",
  "wechat": "zhangsan-seo",
  "email": "zhangsan@example.com",
  "website": "https://your-site.com",
  "qrCodeText": "https://your-wechat-add-url-or-vcard",
  "primaryColor": "#1F4E78",
  "accentColor": "#D9550B",
  "note": "本报告基于免费工具自动生成。深度改造与诊断可加微信咨询。"
}
```

- 留空的字段(`""`)在报告里不显示。
- `qrCodeText` 填**加微信的 URL** 或你的 vCard URL,会自动生成 SVG 二维码,贴在报告底部联系卡上——客户用手机一扫就加上你。
- `primaryColor` / `accentColor` 是十六进制色值,**报告全套配色跟着变**——分发出去自带你的品牌色,不像别人都用同一套蓝色。

**这一步也只做一次,以后所有报告自动带上**。

## 二、跑一次体检

最常见情况——你有客户的网址 `https://example.com`:

```bash
node bin/cli.js https://example.com
```

跑完看到这样的输出:

```
✓ 体检完成,用时 8.4 秒
  页面:24   问题:P0=2  P1=11  P2=18  P3=3
  聚类:5 个   Fan-Out:1240 条候选 query
  建站平台:Shopify

  📄 HTML 报告:report/体检报告.html
  📊 Excel 数据:report/体检数据.xlsx
```

双击打开 `report/体检报告.html`,浏览器里看完整报告。把 `report/` 文件夹整个发给客户就行。

### 几种常见情形

| 你的站长什么样 | 命令 |
|----------------|------|
| 普通 Shopify / WordPress / WooCommerce | `node bin/cli.js https://xxx.com` |
| React/Vue 单页应用(SPA) | `node bin/cli.js https://xxx.com --render` |
| 站很大,想抓深点 | `node bin/cli.js https://xxx.com --max-pages 200` |
| 报告放到别的文件夹 | `node bin/cli.js https://xxx.com --out ./客户A` |
| 想包含子域名 `blog.xxx.com` | `node bin/cli.js https://xxx.com --subdomains` |
| 站点 robots 限制太多,先看看完整情况 | `node bin/cli.js https://xxx.com --ignore-robots` |
| 不想从 sitemap 取种,纯 BFS | `node bin/cli.js https://xxx.com --no-sitemap-seed` |

**判断要不要加 `--render`**:先不加跑一次,如果看到大量页面"词数 < 50、缺 H1",或者只抓到 1–2 页,说明这站是 SPA,加 `--render` 重跑。

## 三、读懂报告的 5 个关键区块

### 1. 报告总览(顶部彩色卡片)

最重要看 **P0 数字**:大于 0 说明站有"会阻塞收录"的硬伤,先解决这个再说别的。

### 2. 问题清单(P0–P3 分级表)

P0 = 致命,先修;P1 = 严重影响排名;P2 = 一般质量;P3 = 优化项。

**直接发给开发**:把这个表的"编码、问题、说明、证据"四列复制出来,开发就有可执行任务。

**外贸独立站的高频 P0 / P1**:
- `no-sitemap` — 没有 sitemap.xml,Google 收录不全。
- `apex-dns-missing` — 不带 www 的域名打不开。
- `spa-empty-shell` — SPA 空壳,Google 看不到内容。
- `duplicate-title` — 多页 title 重复(产品页 / 分类页通病)。
- `no-ga` — 没装 Google Analytics。
- `no-gsc-verify` — 没绑 Google Search Console。

### 3. 站点级体检

DNS、robots、sitemap、平台识别、跟踪接入(GA/GTM/Pixel/GSC)、安全响应头一目了然。**外贸站三件套必装**:GA4 + Google Search Console + Meta Pixel。

### 4. 关键词聚类(决定内容架构)

工具把"内容相近"的页面分到一组,告诉你每组的主题词。

**怎么用**:
- 一个聚类里有 5 个页面争同一个词 → 关键词内耗,留 1 个主推、其余加 canonical 指向它。
- 一个重要主题只有 1 个孤立页 → 需要新增几个相关页面形成"内容枢纽"。
- 聚类显示的"主题"和你期望的不一致 → 说明这些页面在 Google 眼里的主题与你想的不一样,要调整。

### 5. Fan-Out 候选 query(可直接拿去用)

每页生成 6 类共 60–80 条候选 query,**每条前面带 ✓ 或 ✗**:

- **✓** = 这条 query 的关键词已经在页面正文里出现了,搜索引擎理论上能匹配。
- **✗** = 关键词没在页面里——这就是**你该补的内容**。

例如某产品页 "Bluetooth Speaker X1" 整体覆盖率显示 23%(10/44),展开看到:
```
[商业类] 1/8 覆盖
   ✗ Bluetooth Speaker X1 pricing       ← 该补价格段落
   ✗ Bluetooth Speaker X1 alternatives  ← 该补"与同款竞品对比"段落
   ✗ Bluetooth Speaker X1 review        ← 该补用户评价
[导航类] 0/8 覆盖
   ✗ Bluetooth Speaker X1 docs          ← 该补"产品手册"链接
   ✗ Bluetooth Speaker X1 manual        ← 同上
```

**这是整个工具最具体可执行的部分**——把 ✗ 当成 todo 清单:

1. **内容补强清单** — 缺哪条直接补哪条段落 / FAQ / alt。
2. **博客选题** — 长尾类和问答类天然就是博客标题。
3. **外链锚文本素材** — 找博客发外链时,锚文本直接抄这里。
4. **Google Ads 选词** — 信息类做内容、商业类投广告。

## 四、把报告分发给客户

打包整个 `report/` 文件夹发给客户,或:

- **HTML 报告** — 直接发邮件附件,客户用浏览器打开。也可以"另存为 PDF"做正式版。
- **Excel 数据** — 适合发给客户的开发/技术。
- **JSON** — 一般用不上,留给做二次分析。

报告底部的"联系方式"卡片会自动显示 `contact.json` 里你填的微信/邮箱。客户拿到报告,有问题自然来找你。

## 五、常见问题

**Q: 跑 `--render` 报错 "No Chrome found"?**
A: 装一下 Google Chrome,或在命令里指定路径:
```bash
node bin/cli.js https://xxx.com --render --chrome "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
```

**Q: 抓到 1 个页就停了?**
A: 大概率是 SPA,首页 HTML 里没有链接。加 `--render` 重跑。

**Q: 客户说"为什么 Lighthouse 给我 100 分,你说我有问题?"**
A: Lighthouse 只看单页"标签齐不齐",看不到跨页问题。本工具补的就是 Lighthouse 的盲区——给客户看 P0 一栏的具体编码即可。

**Q: 关键词聚类不准?**
A: 调整 `--cluster-threshold`,默认 0.18,越大越严格(聚类更少更纯)。
```bash
node bin/cli.js https://xxx.com --cluster-threshold 0.25
```

**Q: 想让报告全英文版?**
A: 当前是中文版。需要英文版的话,跑这个工具的英文版分支 `seo-audit-cli`(项目里有对应说明)。

**Q: 站点数据会上传吗?**
A: 不会。全程本地运行,只对目标站发 HTTP 请求,数据不出本机。
